# Built phase bundles (`content/built/phases/`)

Each file is generated by **`scripts/build.py`** via **`MapsInstructions`** (same content as **`python scripts/generate_prompt.py --phase <slug>`** when **`skill-config.json`** → **`generate_prompt.assembly_mode`** is **`static`** and this file exists).

**Authoritative phase sources** (edit these, not this folder): `content/parts/phases/<slug>.md`.

Each bundle contains (in order):

1. **Role** — `content/parts/solution-analyst-role.md` (headings demoted).
2. **Phase** — source `content/parts/phases/<slug>.md` only (no embedded role).
3. **Library** — excerpts from **`library_files`** plus **`phase_library[slug]`** in **`skill-config.json`**.
4. **Rules** — `rules/*.md` inlined in order per **`phase_rules`** and **`every_phase_rules`** in **`skill-config.json`** (each entry is the rule file stem without `.md`, same as **abd-skill-builder** `phase_rules`).
5. **Principles** — **`critical-quality-steps.md`** plus optional **`phase_critical_quality_notes`** (**Stage 2+** only). **Stage 1** phases (set-workspace … canonical-context) **omit** this section; `principles.md` already appears under **Library** where configured.
