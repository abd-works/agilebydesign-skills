The Five Families × Five Stages
Agile foundation
Frequent feedback. Cross-functional context, not cross-functional teams. Test-driven and executable. Domain-oriented.
Layered context
Keep the window small — but reshape it as work progresses. Wide and shallow at the solution. Narrow and deep at the story.
Human-in-the-loop
Ask the agent hard questions through validating lenses. Humans review each lens to prevent known failure modes before work moves right.
Whole Solution
wide / shallow
Whole-solution view — outline map and boundaries.
Increment
medium
Full map, domain, UX IA, architecture blueprint.
Sprint
narrow / deeper
Increment slice — AC, mockups, arch templates.
Story
narrowest / deep
Tests first, then running production code.

acceptance-criteria docs/stories/acceptance-criteria-increment-2.md
Story · Create Crowd from COH Model List
WHEN the GM opens the Model Browser, selects one or more Models, and chooses Create Crowd from Selection
THEN a new Crowd is added to the Crowd Repository
AND one Character per Model, each carrying a pre-configured Model Identity
Story · No Models Selected
WHEN no Models are selected in the Model Browser
THEN Create Crowd from Selection is disabled and no Crowd or Character is created
Story · Crowd Creation Succeeds
THEN the new Crowd appears in the Crowd Tree with all Characters as child nodes
AND each node shows the type indicator (Model) for its Identity
