# Impact map — hypotheses (plain text)

## Instructions (for skill maintainers)

Same as impact-map-hypotheses.md. Stakeholder file: filled hypothesis body only (no # headers required; use `Outcome:` as a line label if you need a plain-text section marker).

## Notation

Plain-text shape mirrors the markdown file. Default hook for the impact metric is `by {{impact_N_metric}}`; natural rephrasing is fine if all facts stay explicit.

## Template

When filling: write natural, readable sentences. Keep every placeholder's meaning (actor, impact, impact metric, goal, goal metric on outcome lines; feature and the same impact metric on each build line). In the `then` clause use `then we will` plus the goal as a verb phrase (e.g. grow monthly active players), not "achieve Grow ...". Vary connectors (`by`, `when`, `with ... showing`) so it sounds spoken.

Outcome {{outcome_section_goal_label}}
