# abd-impact-mapping

## Overview

Practice skill for collaborative impact mapping: connect goals to actors, observable behaviour impacts, and deliverable options; optional goal and impact metrics; phased backlog with an **Actor / impact** column matching the tree. Ships **six** templates: hierarchy (`impact-map.md` / `.txt`), ASCII **wall sketch** four-column table (`impact-map-ascii.md` / `.txt`, same semantics as the hierarchy, training-style **Objective \| Persona \| Impact \| Initiative** layout), and hypothesis lines (`impact-map-hypotheses.md` / `.txt`), plus `rules/*.md`, `inputs/abd-answers-retrieval.md`, and an HTML manual under `manual/`.

Use when facilitating discovery, aligning scope to organisational outcomes, or feeding story mapping and prioritisation with a visible outcome spine.

## How it fits together

```ascii
  discovery sources (brief, interviews, outcome statements)
              |
              v
  impact map set: hierarchy + ASCII + hypotheses (same engagement)
              |
              +---> story mapping / thin slicing / backlog
```
