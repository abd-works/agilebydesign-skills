Acceptance criteria (exploration) — plain text template  
(Migrated from agile_bots story_bot exploration behavior conventions.)  
Use the SAME story coverage as acceptance-criteria.md; this file is plain text only.  
  
SOURCE TRACEABILITY (required when AC are grounded in a corpus):  
For EVERY numbered acceptance criterion, add an Evidence line with document name and the most specific location you have: chapter, section heading, page or page range, paragraph,  
slide, line range, URL+anchor, chunk/evidence id, etc. If there is no written source, say so  
(date, meeting, channel). You may use a table after the AC list instead of inline Evidence.

Story: <verb-noun story title>

Domain terms (plain text — same vocabulary as the .md story; no asterisks):  
- <Term> — <short gloss: thing, state, action, or rule this story uses>

(Omit filler; only terms that anchor the AC below.)  
  
Acceptance criteria:  
  
1. WHEN <trigger>  
   THEN <outcome>  
   AND <chained outcome>  
   Evidence: <source> — <chapter/section/page/paragraph/...>  
  
2. WHEN <variant>  
   THEN <only what differs>  
   Evidence: <source> — <location>  
  
3. WHEN <error path>  
   THEN <error behavior>  
   BUT <what does not happen>  
   Evidence: <source> — <location>

Template reference only — do not copy the following into generated project files:  
- 4-9 steps worth of coverage per story (WHEN/AND counting heuristic matches bot scanners).  
- Behavioral language; Given belongs in scenarios, not AC.  
- Channel-specific detail for CLI/Panel/API where relevant.  
- Alternate actors; use AND to chain system reactions.  
- Domain terms: include the Domain terms block per story before Acceptance criteria; plain text here, italics for the same terms in the .md file (emphasize-domain-significant-terms rule).  
- Source evidence: match acceptance-criteria.md — every AC numbered item has Evidence or a table row.
