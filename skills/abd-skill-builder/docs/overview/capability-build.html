Capability 4 of 8

Build from parts

Decompose into library parts — merge into a single AGENTS.md on every change.

Problem
A monolithic instruction file is hard to maintain and causes the model to lose focus. Concepts get duplicated and cross-references go out of date.
Solution
Decompose the skill into parts from a library. The skill invokes a build script whenever parts change to merge them into a single AGENTS.md.

Merge pipeline

  content/parts/                    rules/
      process.md                        *.md  ──┐
      library/  (shared norms, shards)          ├──►  phases/built/<phase>.md
      phases/<phase>.md  (phase body)  ─────────┘     (phase + library + rules)
                                                         │
                                                         ▼
  build.py  ──────────────────────────────────────►  AGENTS.md

What each part does

  • content/parts/library/ — shared norms and reference material that multiple phases pull in. Keeps phase files short and DRY.
  • content/parts/phases/<phase>.md — the phase body: procedure, checklist, specific instructions.
  • rules/*.md — normative rules injected into phases that need them (declared in skill-config.json).
  • phases/built/<phase>.md — the assembled, ready-to-inject bundle for one phase session.
  • AGENTS.md — the single file at repo root containing the full assembled skill instructions.

Running the build

python scripts/base/build.py — reads skill-config.json, merges all parts, writes built phase files and AGENTS.md. Always run after editing any content/parts/, library/, or rules/ file.
