Capability 8 of 8

Tests

Comprehensive, automated test suites covering scripts and AI interactions — skills that stay working across every change.

Problem
Skills don't perform as expected. Changes to the skill constantly break existing functionality in unintended ways.
Solution
Require all scripts and even AI interactions to be supported by comprehensive and automated test suites.

Folder layout

test/
├── fixtures/           ← static input files for deterministic tests
│   ├── skill-config-minimal.json
│   └── sample-output.md
├── test_build.py       ← asserts merged AGENTS.md structure is correct
├── test_scanners.py    ← runs each scanner against known pass/fail fixtures
└── test_workspace.py   ← confirms set_workspace.py reads and writes correctly

What to cover

  • Structure tests — merged bundle contains required sections in correct order.
  • Scanner tests — each scanner flags the known-bad fixture and passes the known-good one.
  • Script tests — helper scripts (set_workspace.py, generate.py) produce expected output given known inputs.

Running tests

pytest test/               # run all suites
pytest test/test_build.py  # run a single suite

Integration with scanners

Scanner bindings in rules/scanners.json can reference on: ["ci"] to run scanner checks as part of the same CI pipeline. The test suite and scanners are complementary: tests verify script logic; scanners verify AI output quality at runtime. See Rules & scanners for how scanners are bound.
