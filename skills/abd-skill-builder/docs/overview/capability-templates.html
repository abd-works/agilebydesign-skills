Templates

Discrete template files define the exact shape the AI must fill — layout stays stable across every run.

Problem
AI output structure varies after every run; downstream tools cannot rely on consistent layout. Key information goes missing.
Solution
Separate layout and data requirements into discrete template files. Each template defines the exact shape the AI must fill, keeping artifact layout stable across every run.

How templates work

┌─────────────────────┐   structure contract   ┌──────────────────────┐
│  templates/…        │ ─────────────────────► │  AI fills output     │
│  (output shape)     │                        │  matching template   │
└─────────────────────┘                        └──────────────────────┘

Template conventions

  • Place templates in templates/ at the skill root, or use .template extension alongside the target file.
  • Mark fill-in zones with a clear placeholder convention — e.g., {{SECTION_NAME}} or <!-- fill: description -->.
  • Include the relevant template in the phase prompt so the model has the shape in context.
  • Reference the template path in skill-config.json if build.py needs to embed it during assembly.

Combining with scanners

A scanner can diff the AI output against the template's required sections and flag any missing or incorrectly structured blocks. See the Rules & scanners page for how to wire a scanner to a rule.
