Skill builder — outline

Builds other skills with the layout and capability described here: merge pipeline, checklists, rules, and scripts.

This skill builds other skills with the layout and capabilities outlined below — merge pipeline, checklists, rules, and scripts. Each card links to a detail page. "The skill" means the skill built by the skill builder, not the builder itself.

Capabilities

Workspace & configuration

Problem

Skill writes in the wrong area; reads and writes wrong files. Doesn't pick up your last work.

Solution

A single skill-config.json declares the active workspace so every script and instruction in your skill knows where to conduct its work.

View detail →

Phases & process files

Problem

Guidance gets lost in skills that are multi-step and require workflow.

Solution

Define an ordered pipeline in a single file with references to distinct phase files — one per phase. Each phases/<phase>.md holds inputs, outputs, and procedure for a step. Define steps as Human, AI, or script.

View detail →

Activity checklists

Problem

AI losing track of skill progress; skill steps skipped, executes steps in wrong order. Skill restart is painful after time away.

Solution

Create checklists for the overall process and/or each phase. Update checklist — To Do / IP / Done — as skill work progresses.

View detail →

Build from parts

Problem

A monolithic instruction file is hard to maintain and causes the model to lose focus. Concepts get duplicated and cross-references go out of date.

Solution

Decompose the skill into parts from a library. The skill invokes a build script whenever parts change to merge them into a single AGENTS.md.

View detail →

Prompt generation & injection

Problem

AI picks and chooses what parts of the skill to follow. Templates and guidance become optional. AI requires constant reminding on how to use the skill effectively.

Solution

The skill invokes a generate script on every call which generates the parts most relevant to the request and injects them directly into the AI chat — forcing attention on context, guidance, templates, and rules. The model gets what it needs, nothing more.

View detail →

Rules & scanners

Problem

AI stubbornly makes the same mistakes despite the best prompting. Guidance gets scattered across the skill over time. AI constantly requires close supervision to get to quality.

Solution

Decompose rules into separate files, guide AI by injecting rules into prompts, enforce rules mechanically through scanners — scripts that check for syntactic smells and act as x-ray vision for AI slop.

View detail →

Templates

Problem

AI output structure varies after every run; downstream tools cannot rely on consistent layout. Key information goes missing.

Solution

Separate layout and data requirements into discrete template files. Each template defines the exact shape the AI must fill, keeping artifact layout stable across every run.

View detail →

Tests

Problem

Skills don't perform as expected. Changes to the skill constantly break existing functionality in unintended ways.

Solution

Require all scripts and even AI interactions to be supported by comprehensive and automated test suites.
