Capability 2 of 8

Phases & process files

An ordered pipeline with distinct phase files — every step defined for Human, AI, or script.

Problem
Guidance gets lost in skills that are multi-step and require workflow.
Solution
Define an ordered pipeline in a single file with references to distinct phase files — one per phase. Each phases/<phase>.md holds inputs, outputs, and procedure for a step. Define steps as Human, AI, or script.

File structure

┌─────────────┐   defines   ┌────────────────────────────────────┐
│ process.md  │ ──────────► │  Phase 1 → Phase 2 → Phase 3 → …   │
└─────────────┘             └────────────────────────────────────┘
                                      │ each phase
                                      ▼
                              ┌──────────────────────┐
                              │  phases/<phase>.md   │
                              │  I/O, procedure,     │
                              │  TOC: AI vs code     │
                              └──────────────────────┘

What goes where

  • process.md — high-level pipeline table. Each row: phase name, runner (AI / code), link to phase file. Consumed by build.py to know which phases exist.
  • phases/<phase>.md — one file per phase. Sections: Inputs, Outputs, Procedure, optional Action Checklist. Long norms are pulled from library/ to keep phase files short.
  • skill-config.json → phase_files — ordered list of phase slugs; drives both generate.py and build.py.
