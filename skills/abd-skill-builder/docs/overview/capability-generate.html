Capability 5 of 8

Prompt generation & injection

Generated on every call — context, guidance, templates, and rules injected directly into the AI chat.

Problem
AI picks and chooses what parts of the skill to follow. Templates and guidance become optional. AI requires constant reminding on how to use the skill effectively.
Solution
The skill invokes a generate script on every call which generates the parts most relevant to the request and injects them directly into the AI chat — forcing attention on context, guidance, templates, and rules. The model gets what it needs, nothing more.

How it works

│   User   │ ─────────────► │ generate.py  │ ────────────────────► │  AI Chat │
│  / Agent │                │              │                       │          │
└──────────┘                └──────────────┘                       └──────────┘
                                    │
                                    ▼
                        ┌──────────────────────┐
                        │  content/parts/       │
                        │  phases/  library/    │
                        └──────────────────────┘

Usage

python scripts/base/generate.py --phase <slug>

  • Looks up the phase in skill-config.json → phase_files.
  • Reads the phase file from content/parts/phases/<slug>.md.
  • Inlines library shards and rules declared for that phase.
  • Outputs the assembled bundle (typically copied into the AI chat as the session opener).
  • Creates missing workspace checklists under progress/ on first run for this phase.

Why not just use AGENTS.md?

AGENTS.md is the full, compiled skill — useful for Cursor agent mode where the full context is always present. generate.py is for chat-based sessions where you want laser focus on one phase at a time and cannot rely on the agent having read the full file.
